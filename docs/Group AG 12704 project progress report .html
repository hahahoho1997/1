Problems Encountered
➢ When using MQTT to receive data, the data is not the same as the original one. The initial
guess is MQTT protocol has limited the string size, so it cannot pass numbers with large
values.
➢ When using message.payload.decode() function to decode the data from two channels,
different datasets cannot print out and read separately. The solution we figure out now is to
create two python files. One file deals with temperature data. Another file deals with water
level data, then run two files concurrently. However, maybe there are more clever ways to
separate the datasets.
➢ The water level sensor is not too accurate. Sometimes when the water level is low, the
reading shows the water level is high, which means we only can set up water level boundary
approximately.
